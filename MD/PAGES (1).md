# 🛠️ IMPLEMENTATION_SEQUENCE.md — ลำดับการทำโปรเจกต์จากเริ่มจนจบ

เอกสารนี้ใช้เป็นแผนงานลำดับการพัฒนาโดยเริ่มจากการควบคุมเครื่องด้วย AP Mode ก่อนเสมอ แล้วค่อยขยายไปสู่ Online Mode และระบบระยะไกลต่อไป

---

## 1. ทำ AP Mode ให้ทำงานก่อน (ต้องทำก่อนเสมอ)

- สร้างโหมด AP Mode บน ESP8266 ให้ตัวเครื่องปล่อย Wi-Fi ของตัวเอง
- ตั้งค่า SSID และ Password ของ AP Mode ให้ใช้งานง่ายและปลอดภัย
- เสิร์ฟหน้าเว็บจากตัวเครื่องที่ IP `192.168.4.1`
- ทำให้หน้าเว็บนี้สามารถควบคุมเครื่องได้ทันที เช่น ให้อาหาร, ตั้งเวลา, ดูสถานะ, ชั่งน้ำหนัก, ดูประวัติ
- สร้าง QR Code สำหรับ AP Mode โดยบรรจุข้อมูล SSID และ Password ของ AP ของเครื่อง
- ทดสอบว่าเมื่อสแกน QR Code แล้วมือถือจะเชื่อมต่อ Wi-Fi ของเครื่องได้ทันทีและเปิดหน้า AP Mode ได้เลยโดยไม่ต้องพิมพ์รหัส

> จุดสำคัญ: AP Mode เป็นจุดเริ่มต้นของการควบคุมเครื่องและต้องทำให้สมบูรณ์ก่อนแล้วค่อยขยายต่อ

---

## 2. ทำหน้าเว็บเดียวกันสำหรับ AP Mode และ Online Mode

- ออกแบบหน้าเว็บให้ใช้โครงสร้างเดียวกันทั้งสองโหมด
- ฟังก์ชันควบคุมที่เหมือนกัน เช่น Manual Feed, Schedule, Calibration, Status, History ต้องใช้หน้าเดียวกัน
- โหมด AP Mode และ Online Mode ต่างกันแค่ช่องทางเชื่อมต่อเท่านั้น ไม่ต่างกันในฟังก์ชันควบคุม
- ทำให้ผู้ใช้ไม่ต้องเรียนรู้หน้าจอใหม่เมื่อเปลี่ยนโหมด

---

## 3. ทดสอบการทำงานแบบ Local ก่อน

- ทดสอบการควบคุมจาก AP Mode โดยตรงผ่าน Wi-Fi ของเครื่อง
- ตรวจว่าเครื่องตอบสนองคำสั่งได้ทันที
- ตรวจว่า LED, น้ำหนัก, ประวัติการให้อาหาร และตารางเวลาแสดงผลถูกต้อง
- ตรวจว่าหน้าเว็บใน AP Mode ทำงานเหมือนกับ Online Mode

---

## 4. เพิ่ม Online Mode โดยใช้ Wi-Fi จากที่อื่น

- เพิ่มให้เครื่องเชื่อมต่อ Wi-Fi จากบ้านหรือสถานที่อื่น (STA Mode)
- เมื่อเชื่อมต่อสำเร็จให้เครื่องเข้าสู่ Online Mode และพร้อมใช้เครือข่ายอินเทอร์เน็ต
- คงให้ใช้หน้าเว็บเดียวกันกับ AP Mode เพื่อควบคุมและดูสถานะต่อไป

---

## 5. ทำการเชื่อมต่อกับ MQTT / HiveMQ

- ตั้งค่าตัวกลางส่งคำสั่งและสถานะแบบเรียลไทม์ผ่าน MQTT
- ให้ ESP8266 Subscribe คำสั่งจาก topic ของตัวเอง
- ให้ ESP8266 Publish สถานะกลับไปยัง topic ที่เหมาะสม
- ทำให้คำสั่งจากเว็บทั้ง AP Mode และ Online Mode ส่งถึงเครื่องได้แบบเดียวกัน

---

## 6. ทำการเชื่อมต่อกับ Firebase สำหรับข้อมูลถาวร

- เก็บประวัติการให้อาหาร ตารางเวลา และข้อมูลเครื่องใน Firebase
- ทำให้ข้อมูลที่บันทึกในเครื่องสามารถซิงค์ขึ้น Firebase ได้เมื่อออนไลน์
- รองรับการเริ่มต้นจาก AP Mode ก่อน แล้วค่อยใช้ Online Mode ในขั้นตอนต่อไป

---

## 7. ทำ Remote Dashboard / การควบคุมจากระยะไกล

- ทำหน้าเว็บสำหรับการควบคุมจากระยะไกลผ่าน Online Mode
- ใช้หน้าเว็บเดียวกับ AP Mode เพื่อให้ไม่ต้องแยก UI
- ทำให้ผู้ใช้สามารถดูสถานะและควบคุมเครื่องได้จากทุกที่

---

## 8. ทำระบบความปลอดภัยและการตั้งค่าเครื่อง

- ตั้งค่า Device ID / Password หรือ Token สำหรับการเข้าถึงเครื่อง
- ใช้ AP Mode สำหรับการตั้งค่าเบื้องต้นและการเข้าถึงง่าย
- ค่อยเพิ่มข้อจำกัดความปลอดภัยใน Online Mode เมื่อระบบพร้อมใช้งานจริง

---

## 9. ทดสอบครบทั้งสองโหมด

- ทดสอบจาก AP Mode: เชื่อมต่อผ่าน QR Code และควบคุมเครื่องได้ทันที
- ทดสอบจาก Online Mode: ใช้ Wi-Fi จากที่อื่นและควบคุมเครื่องได้เหมือนกัน
- ตรวจว่า UI, ฟังก์ชัน และสถานะทั้งหมดแสดงผลและทำงานเหมือนกัน

---

## 10. พร้อมใช้งานจริง

- ตรวจสอบว่าโหมด AP Mode ยังสามารถใช้ได้แม้ไม่มีอินเทอร์เน็ต
- ตรวจสอบว่า Online Mode ทำงานได้เมื่อเชื่อมต่อ Wi-Fi จากภายนอก
- ให้ผู้ใช้ใช้ AP Mode เป็นทางเข้าแรกสุดสำหรับการตั้งค่าและควบคุมเบื้องต้น พร้อมกับ QR Code ที่ง่ายต่อการใช้งาน
