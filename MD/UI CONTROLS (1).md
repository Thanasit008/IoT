# 📖 SYSTEM_GUIDE.md — คู่มืออธิบายการทำงานของเครื่องและเว็บไซต์

เครื่องเป็น **เครื่องให้อาหารปลาอัตโนมัติ** มีมอเตอร์ปล่อยอาหารและเซนเซอร์ชั่งน้ำหนัก ควบคุมผ่านชิป **ESP8266** ซึ่งเชื่อมกับเว็บไซต์ได้ 2 ทาง — **AP Mode** (ต่อ Wi-Fi ของตัวเครื่องเอง ไม่ต้องมีเน็ต) และ **Online Mode** (เครื่องต่อ Wi-Fi จากแหล่งอื่น เช่น Wi-Fi บ้าน เชื่อม **HiveMQ** สำหรับสั่งงาน/รับสถานะแบบเรียลไทม์ และซิงค์ประวัติเข้า **Firebase** แล้วผู้ใช้ควบคุมผ่านเว็บที่โฮสต์บน **Vercel** จากที่ไหนก็ได้) **ระบบนี้ไม่มี Login/สมัครสมาชิก** ผู้ใช้เข้าคุมเครื่องด้วยการกรอก **Device ID + Password** ของเครื่องนั้นโดยตรง

---

## 🐟 เครื่องทำอะไรได้บ้าง — และทำงานอย่างไร

**1. ปล่อยอาหาร** — ปล่อยได้ทั้งแบบกดสั่งทันที (Manual) และตั้งเวลาอัตโนมัติ (สูงสุด 4 รอบ/วัน) เมื่อได้รับคำสั่ง ESP8266 สั่งมอเตอร์หมุน พร้อมให้ **Load Cell** วัดน้ำหนักที่ปล่อยออกมาต่อเนื่อง เมื่อน้ำหนักถึงเป้าหมายมอเตอร์จะหยุดทันที — ยึดน้ำหนักจริงเป็นหลัก ไม่ใช่กะเวลา

**2. จับเวลาแม้ไม่มีอินเทอร์เน็ต** — โมดูล **RTC DS3231** เป็นนาฬิกาในตัวที่เดินเวลาต่อเนื่องด้วยแบตเตอรี่สำรอง ไม่พึ่งอินเทอร์เน็ตหรือ HiveMQ/Firebase ทำให้ให้อาหารตรงเวลาได้เสมอ

**3. ชั่งน้ำหนักและคำนวณอาหารคงเหลือ** — Load Cell + **HX711** อ่านน้ำหนักถังตลอดเวลา นำมาหารด้วยปริมาณที่ใช้เฉลี่ยต่อวัน ได้ "จำนวนวันที่ใช้ได้อีก" แล้วแสดงสถานะ 🟢 (มากกว่า 7 วัน) 🟡 (น้อยกว่า 7 วัน) 🔴 (น้อยกว่า 3 วัน)

**4. เชื่อมต่อเครือข่าย 2 แบบ** — ตอนเปิดเครื่องครั้งแรกหรือเชื่อมต่อ WiFi บ้านไม่สำเร็จ เครื่องสร้าง WiFi ของตัวเอง (Access Point) ให้ตั้งค่าที่ `192.168.4.1` เมื่อเชื่อมต่อ WiFi บ้านสำเร็จ เครื่องจะเชื่อมต่อ **HiveMQ (MQTT Broker)** ทำให้เว็บ Online Mode ส่งคำสั่งและรับสถานะแบบเรียลไทม์จากทุกที่ ส่วนประวัติจะถูกซิงค์เข้า Firebase แยกต่างหาก

**5. บอกสถานะผ่านไฟ LED** — 🔴 แดง = เปิดเครื่องแล้ว, 🟡 เหลือง = โหมด AP Mode รอตั้งค่า, 🟢 เขียว = เชื่อมต่อ HiveMQ สำเร็จพร้อมควบคุมระยะไกล

**6. บันทึกข้อมูลแม้ไม่มีเน็ต** — ทุกครั้งที่ให้อาหารสำเร็จ เครื่องบันทึกข้อมูล (วันที่ เวลา ปริมาณ โหมด น้ำหนักก่อน-หลัง) ไว้ในตัวเครื่องก่อนเสมอ แล้วซิงค์ขึ้น **Firebase Database** ทันทีที่มีอินเทอร์เน็ต

---

## 🖥️ เว็บไซต์ทำอะไรได้บ้าง — และทำงานอย่างไร

**1. ตั้งค่าเครื่องครั้งแรก (AP Mode: `192.168.4.1`)** — เว็บหน้านี้เสิร์ฟจากตัว ESP8266 เอง ไม่ผ่านอินเทอร์เน็ต ใช้ตั้งชื่อ/รหัสผ่าน WiFi ของเครื่องเอง และเลือก WiFi บ้านให้เชื่อมต่อ กรอกแล้วกดบันทึก เครื่องจะรีสตาร์ทและลองเชื่อมต่อทันที; ใน AP Mode ควรมี QR Code เพื่อเชื่อมต่อ Wi-Fi ของเครื่องและเปิดหน้าเว็บได้ทันทีโดยไม่ต้องพิมพ์รหัส

**2. ควบคุมเครื่องแบบ AP Mode ได้ทันที** — ตราบใดที่ยังเชื่อมต่อ WiFi ของเครื่องอยู่ สามารถสั่งให้อาหาร ตั้งเวลา ดูน้ำหนัก ดูประวัติได้เลย ไม่ต้องกรอก Device ID/Password ซ้ำเพราะอยู่ในวงเครือข่ายเดียวกับเครื่องอยู่แล้ว

**3. เข้าถึงเครื่องด้วย Device ID + Password (Online Mode)** — ไม่มีระบบ Login/สมัครสมาชิก ผู้ใช้กรอก Device ID + Password ของเครื่องที่ต้องการคุม เว็บส่งไปตรวจสอบผ่าน **Firebase Cloud Function** ถ้าถูกต้องจะได้ **Token ชั่วคราว** สำหรับคุมเครื่องนั้น และเว็บจะจำ Device ID ไว้ใน localStorage ของเบราว์เซอร์ให้อัตโนมัติในครั้งถัดไป (ไม่ sync ข้ามอุปกรณ์)

**4. Dashboard ควบคุมหลัก (Online Mode, โฮสต์บน Vercel)** — กดปุ่มบนเว็บ คำสั่งจะถูก **Publish ผ่าน HiveMQ** ไปยัง ESP8266 ที่ออนไลน์อยู่แทบจะทันที เครื่องทำงานเสร็จแล้วส่งสถานะกลับผ่าน HiveMQ เช่นกัน ทำให้หน้าเว็บอัปเดตแบบเรียลไทม์โดยไม่ต้องรีเฟรช ส่วนประวัติที่บันทึกเสร็จแล้วจะถูกดึงมาแสดงจาก Firebase

**5. สั่งงานด้วยเสียง** — ใช้ **Web Speech API** ของเบราว์เซอร์แปลงเสียงเป็นข้อความ ใส่ในช่องคำสั่งให้ตรวจสอบก่อนกดส่งจริง (ต้องใช้งานผ่าน HTTPS ซึ่ง Vercel รองรับให้อัตโนมัติ)

**6. บันทึกและแสดงประวัติ** — ข้อมูลจากเครื่องถูกซิงค์เข้า Firebase แบบ Real-time เว็บไซต์ดึงมาแสดงเป็นตารางที่กรองตามช่วงวันที่ได้ทันที

## 🔗 ภาพรวมการเชื่อมโยงทุกส่วน

```mermaid
flowchart LR
    ผู้ใช้ -->|กรอก Device ID+Password| เว็บไซต์["เว็บไซต์ (Vercel)"]
    เว็บไซต์ -->|ตรวจสอบ| CloudFn[Firebase Cloud Function]
    CloudFn -->|Token ชั่วคราว| เว็บไซต์
    เว็บไซต์ -->|Local: WiFi ตรง| ESP8266
    เว็บไซต์ <-->|Remote: MQTT| HiveMQ[(HiveMQ Broker)]
    HiveMQ <-->|คำสั่ง/สถานะ| ESP8266
    ESP8266 -->|สั่งมอเตอร์| มอเตอร์ปล่อยอาหาร
    ESP8266 <-->|อ่าน/เขียนเวลา| RTC[RTC DS3231]
    ESP8266 <-->|อ่านน้ำหนัก| LoadCell[Load Cell + HX711]
    ESP8266 -->|บันทึกประวัติถาวร| Firebase[(Firebase Database)]
    เว็บไซต์ -->|อ่านประวัติด้วย Token| Firebase
    LoadCell -->|คำนวณวันคงเหลือ| แจ้งเตือน
```

## 📌 สรุปใจความสำคัญ

| ส่วน | หน้าที่หลัก | กลไกเบื้องหลัง |
|---|---|---|
| ESP8266 | สมองกลางของเครื่อง | รับคำสั่ง สั่งมอเตอร์ เชื่อมต่อ HiveMQ + Firebase |
| RTC DS3231 | จับเวลาให้อาหารอัตโนมัติ | นาฬิกาในตัว ไม่พึ่งอินเทอร์เน็ต |
| Load Cell + HX711 | ชั่งน้ำหนักอาหาร | วัดแบบ Real-time เพื่อควบคุมปริมาณและคำนวณวันคงเหลือ |
| เว็บ AP Mode (`192.168.4.1`) | ตั้งค่าเครื่อง + ควบคุมใกล้เครื่อง | เสิร์ฟจาก ESP8266 เอง ไม่ต้องกรอก Password ซ้ำ/ไม่ต้องมีเน็ต และให้ใช้ QR Code เพื่อเชื่อมต่อได้เร็วขึ้น |
| เว็บ Online Mode (Vercel) | ควบคุมจากที่ไหนก็ได้ | กรอก Device ID+Password ผ่าน Cloud Function + สั่งงานผ่าน HiveMQ + อ่านประวัติจาก Firebase |
| HiveMQ | ส่งคำสั่ง/สถานะแบบเรียลไทม์ | MQTT Broker เชื่อมเว็บกับเครื่อง (1 Cluster ใช้ร่วมกันทั้งระบบ) |
| Firebase | เก็บข้อมูลถาวร + ตรวจสอบสิทธิ์ | Database + Cloud Function + Security Rules (1 Project ใช้ร่วมกันทั้งระบบ) |

**หมายเหตุ:** ไม่มีระบบบัญชีผู้ใช้ในระบบนี้ ความปลอดภัยทั้งหมดขึ้นอยู่กับความยากง่ายของ Password ที่ตั้งให้แต่ละเครื่อง — ควรตั้งให้สุ่มและยาวพอสมควรตอนตั้งค่าเครื่องครั้งแรก
